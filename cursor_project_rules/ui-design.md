# UI 设计规范

本文档记录了项目中的 UI 设计原则和规范。

## 文本颜色规范

为了保证界面的清晰度和对比度，文本颜色应遵循以下原则：

- **拒绝灰色文本**：避免使用“灰不拉几”的中间色调（如 `text-neutral-400`, `text-neutral-500` 等作为主要文本颜色）。
- **极简对比**：主要文本颜色要么是**纯黑 (`text-black`)**，要么是**纯白 (`text-white`)**。
- **背景适配**：
  - 在浅色背景上，使用黑色文本。
  - 在深色背景上，使用白色文本。
- **表头与内容**：
  - 表格头部（TableHeader）建议使用深色背景配白色文本，以突出结构。
  - 表格内容（TableBody）建议使用浅色背景配黑色文本，以保证可读性。

## 交互状态

- **Hover 状态**：按钮或其他交互元素在 hover 后，其文本颜色应保持清晰（通常保持白色或黑色不变），不要变成模糊的灰色。

## 布局规范

- **边距控制**：顶部边距不宜过大，保持紧凑的视觉结构（如使用 `p-4` 而不是 `p-6` 或更大的 padding）。
